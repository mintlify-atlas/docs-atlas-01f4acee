The Relationship Visualizer provides two worksheets dedicated to troubleshooting: `diagnostics` and `console`. Together they give you a full picture of the environment your workbook is running in and the output Graphviz produces when it renders your graphs.

## The `diagnostics` worksheet

The `diagnostics` worksheet reports key facts about the workbook and the system it is running on. Open it whenever you need to confirm your setup or gather information for a bug report.

You access it from the **Launchpad** ribbon tab.

<Steps>
  <Step title="Open the Launchpad tab">
    Click the **Launchpad** tab in the Excel ribbon.
  </Step>
  <Step title="Open the diagnostics worksheet">
    In the **Maintenance** group, click the **diagnostics** toggle button.
  </Step>
</Steps>

The worksheet displays the following information:

- **Workbook file name** — the full path to the current Excel file
- **Operating system** — Windows 64-bit, Windows 32-bit, or macOS
- **Graphviz version** — the version of Graphviz that the Relationship Visualizer detected and will call when rendering graphs
- **Temporary file directories** — paths used to store intermediate files during graph generation
- **Image cache directories** — paths used by the `style designer` to store font preview images and color swatches

<Tip>
If you are filing a bug report, copy the contents of the `diagnostics` worksheet and include it. It gives the author everything needed to reproduce your environment.
</Tip>

### Diagnostics ribbon tab

The `diagnostics` worksheet has its own ribbon tab with three groups.

**Diagnostics group**

| Control | Type | Description |
|---|---|---|
| **Refresh** | Button | Forces a fresh read of all diagnostic values. Use this after installing a new version of Graphviz or changing your system `PATH` to confirm the workbook picks up the change. |

**Image Cache group**

The **Style Designer** generates preview images for fonts and colors and saves them to disk. The **Image Cache** group lets you delete those cached images so they are regenerated on next use.

| Control | Type | Description |
|---|---|---|
| **Delete colors** | Button | Deletes the color-swatch images used in the `style designer` color dropdowns. |
| **Delete fonts** | Button | Deletes the font preview images generated by Graphviz for the `style designer` font gallery. |

<Warning>
Cached images are held in memory if you opened the `style designer` worksheet before pressing these buttons. Deleting the disk files alone will not force regeneration until you close and reopen the workbook.
</Warning>

**Help group**

| Control | Type | Description |
|---|---|---|
| **Help** | Button | Opens the diagnostics documentation page in your browser. |

## The `console` worksheet

The `console` worksheet captures the output of every `dot` command the Relationship Visualizer issues. Rows with a shaded background show the `dot` command that was run; rows with a white background show the messages `dot` returned.

You access it from the **Launchpad** ribbon tab or from the **Debug** group on the **Graphviz** ribbon tab.

<Steps>
  <Step title="Open the console worksheet">
    On the **Launchpad** tab, click **console** in the **Graphviz dot** group. Alternatively, click **Console** in the **Debug** group on the **Graphviz** tab.
  </Step>
  <Step title="Refresh your graph">
    Switch back to the `data` worksheet and click **Refresh** (or press the keyboard shortcut). The `console` worksheet updates with the output from that `dot` invocation.
  </Step>
</Steps>

### Console ribbon tab

The `console` worksheet has its own ribbon tab with four groups.

**Console Switches group**

| Control | Type | Description |
|---|---|---|
| **Log to console** | Checkbox | Turns logging of `dot` command output on or off. Uncheck this if you want to run silently with no output captured. |
| **Run Graphviz in verbose mode** | Checkbox | Appends the `-V` flag to the `dot` command. Graphviz then emits detailed diagnostic messages describing each processing step. |
| **Append console messages** | Checkbox | When checked, each `dot` invocation appends its output to the existing log. When unchecked, each invocation replaces the previous output so you always see only the most recent run. |

**Console Text group**

| Control | Type | Description |
|---|---|---|
| **Clear** | Button | Erases all content from the `console` worksheet. |
| **Save to File** | Button | Opens a file save dialog so you can write the console output to a text file. |
| **Copy to Clipboard** | Button | Copies the full console content to the clipboard. **Windows only.** |

**Critical Messages group**

This group contains toggle buttons for enabling independent message-routing channels. You can combine them freely:

| Control | Type | Description |
|---|---|---|
| **Log to console** | Toggle button | Enables or disables logging of `dot` messages to the `console` worksheet. |
| **Run Graphviz in verbose mode** | Toggle button | Enables or disables the `-V` verbose flag on `dot` invocations. |
| **Append console messages** | Toggle button | Controls whether output is appended to or replaces the existing log. |

**Help group**

| Control | Type | Description |
|---|---|---|
| **Help** | Button | Opens the console documentation page in your browser. |

### Enabling verbose mode

Standard `dot` output includes warnings about problematic data, such as node labels that are wider than their containing cells. Verbose mode adds a detailed trace of the entire rendering pipeline, which is useful when:

- A graph renders blank or partially
- You see unexpected layout behavior
- You need to confirm which version of `dot` is being called
- You are troubleshooting a style or attribute that seems to be ignored

To enable verbose mode, check **Run Graphviz in verbose mode** in the **Console Switches** group, then refresh your graph. The `console` worksheet will contain a full trace of what Graphviz processed.

### Reading warning messages

Common messages you may see in the `console` worksheet:

| Message pattern | Meaning |
|---|---|
| `Warning: node ... width ... too small for label` | The node's explicit `width` attribute is narrower than the rendered label. Increase the `width` value or reduce the label length. |
| `Error: ... syntax error near line ...` | The DOT source contains a syntax error. Check the `source` worksheet at the indicated line number. |
| `Warning: ... is an attribute of ... not ...` | An attribute was applied to the wrong element type (for example, a node-only attribute placed on an edge). |
| `Error: could not find or load ...` | An image file referenced in the graph could not be located. Check the **Image Path** setting in the `settings` worksheet. |

<Note>
All three message-routing toggles in the **Critical Messages** group operate independently. You can disable all of them for fully silent operation, enable all for maximum diagnostic output, or mix and match depending on your workflow.
</Note>
