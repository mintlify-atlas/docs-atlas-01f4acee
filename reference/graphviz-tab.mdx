The **Graphviz** ribbon tab is your primary interface for generating and controlling diagrams. It appears automatically whenever you select the `data`, `graph`, `styles`, `settings`, or `about…` worksheet. Every button, toggle, and dropdown on the tab is listed and explained here.

<Note>
The **Graphviz** tab is context-sensitive. It only activates when one of the supported worksheets is selected. Switching to an unrelated worksheet hides the tab.
</Note>

## Visualize group

The **Visualize** group contains the controls for rendering your graph inside the workbook.

| Control | Type | Description |
|---|---|---|
| **Refresh** | Button | Sends your Excel data through Graphviz and displays the resulting graph in the workbook. |
| **Automatic** | Toggle button | When pressed, the workbook monitors keystrokes and automatically refreshes the graph whenever a cell value changes. Also requires **Worksheet** to be set to `data`. |
| **Apply Styles** | Toggle button | When pressed, applies the style-attribute strings from the `styles` worksheet to matching nodes, edges, and clusters. Unpress to render with default Graphviz formatting. |
| **Apply Attributes** | Toggle button | When pressed, includes any raw Graphviz attributes entered in the `Attributes` column of the `data` worksheet. Unpress to omit them entirely. |
| **Zoom In** | Button | Increases the displayed graph scale by 5%. |
| **Zoom Out** | Button | Decreases the displayed graph scale by 5%, useful for viewing large graphs without scrolling. |
| **Current Zoom** | Text display | Shows the current magnification level. The range is 5% to 150% in 5% increments. |
| **View** | Dropdown | Selects which view column from the `styles` worksheet controls which nodes and edges are visible in the diagram. |
| **Image Type** | Dropdown | Sets the image format used when displaying the graph inside the workbook: `bmp`, `gif`, `jpg`, `png`, or `svg`. SVG display requires Office 365. |
| **Worksheet** | Dropdown | Controls where the graph image is placed: `data` (to the right of your data columns) or `graph` (on the dedicated `graph` worksheet). |

<Note>
When **Automatic** is enabled, the graph redraws as you type. This is convenient during editing but can slow performance on large data sets. Toggle it off before making bulk edits.
</Note>

## Publish group

The **Publish** group writes graph output to files on disk rather than displaying it inside the workbook.

| Control | Type | Description |
|---|---|---|
| **Publish** | Button | Runs Graphviz and writes the resulting image to a file in the configured directory. |
| **Publish all views** | Button | Iterates over every view defined in the `styles` worksheet and writes a separate output file for each one. |
| **Get Directory** | Button | Opens a directory picker. Once a directory is chosen, its path replaces the button label. |
| **File Prefix** | Edit box | The base portion of the output file name, for example `Graph`. Supports run-time substitution tokens: `%D` (date), `%T` (time), `%E` (layout engine), `%S` (splines), `%V` (view name), `%W` (worksheet name). |
| **File Format** | Dropdown | Output file format: `bmp`, `gif`, `jpg`, `pdf`, `png`, `ps`, `svg`, or `tiff`. |
| **Filename options** | Dropdown | A checklist of options that control what gets appended to the file name at run time. |
| **Add date/time to the filename** | Checkbox | When checked, appends the current date and time to the file name. |
| **Add Layout/Splines to the filename** | Checkbox | When checked, appends the active layout engine and spline setting to the file name. |

<Tip>
Use the `%V` token in the **File Prefix** box together with **Publish all views** to generate a uniquely named file per view in a single operation.
</Tip>

## Graph Layout group

The **Graph Layout** group selects the Graphviz layout engine. The buttons behave like radio buttons — only one engine is active at a time.

| Engine | Best for |
|---|---|
| `dot` | Hierarchical layouts — org charts, dependency trees, directed flows |
| `fdp` | Force-directed undirected graphs — moderate-sized networks |
| `sfdp` | Scaled force-directed — very large undirected networks |
| `twopi` | Radial layouts — hub-and-spoke diagrams |
| `neato` | Spring-model undirected graphs — small-to-medium networks |
| `circo` | Circular layouts — ring and cycle topologies |
| `osage` | Array/cluster layouts — Kanban boards, categorical groupings |
| `patchwork` | Squarified treemaps — proportional area comparisons |

## Splines group

The **Splines** group controls how edges are routed between nodes. The buttons behave like radio buttons.

| Button | Description |
|---|---|
| **false** | Edges are drawn as straight lines. |
| **true** | Edges use a combination of straight segments and free-flowing curves. |
| **curved** | Edges are drawn as smooth, continuous curves. |
| **ortho** | Edges are routed with horizontal and vertical segments at 90-degree bends. |
| **polyline** | Edges use straight segments with angular bends (not restricted to right angles). |
| **none** | Edges are not drawn, but still influence node placement. |

## Graph Type group

The **Graph Type** group determines whether the diagram is directed or undirected. The buttons behave like radio buttons.

| Button | Description |
|---|---|
| **undirected** | Creates an undirected graph. Edges have no direction and are drawn without arrowheads. |
| **directed** | Creates a directed graph (digraph). Edges have a defined direction and are drawn with arrowheads. |

## Drawn First group

The **Drawn First** group controls the `outputorder` attribute — the sequence in which Graphviz draws nodes and edges. The buttons behave like radio buttons.

| Button | Description |
|---|---|
| **breadth** | Draws edges before nodes. Nodes appear on top of edges. |
| **nodes** | Draws nodes first; edges are drawn afterward and appear on top. |
| **edges** | Draws edges first; nodes are drawn afterward and appear on top of edges. |

## Layout Options group

The **Layout Options** group changes dynamically based on the selected layout engine. Options shown are specific to the active engine.

<AccordionGroup>
  <Accordion title="dot options">
    The `dot` engine provides the most layout options.

    **Rank Direction** — determines the flow direction of the graph:

    | Button | Description |
    |---|---|
    | **tb** | Top-to-bottom. Ranks flow downward (default). |
    | **bt** | Bottom-to-top. Ranks flow upward. |
    | **lr** | Left-to-right. Ranks flow horizontally. |
    | **rl** | Right-to-left. Ranks flow horizontally in reverse. |

    **Edge ordering** — controls how edges are arranged around each node:

    | Button | Description |
    |---|---|
    | **in** | Preserves the order of incoming edges. |
    | **out** | Preserves the order of outgoing edges. |

    **New Rank** — when enabled, Graphviz computes a single global ranking across all clusters instead of ranking each cluster independently. Useful when you want tightly coordinated layouts across clustered subgraphs.

    **Compound** — when enabled, edges can connect to cluster boundaries using `lhead` and `ltail` attributes.

    | Value | Description |
    |---|---|
    | **false** | Edges cannot attach to cluster boundaries. |
    | **true** | Edges can enter or leave clusters at their boundaries. |

    **Cluster Rank** — controls how clusters are ranked relative to each other:

    | Value | Description |
    |---|---|
    | **local** | Each cluster is ranked independently (compact cluster layouts). |
    | **global** | All clusters participate in a single unified ranking. |
  </Accordion>

  <Accordion title="fdp options">
    **Overlap** — controls how Graphviz resolves node collisions:

    | Value | Description |
    |---|---|
    | **compress** | Compresses the layout after overlap removal for a tighter result. |
    | **prism** | Uses a stress-based algorithm to separate nodes while preserving structure. |
    | **scale** | Uniformly scales the layout until nodes no longer overlap. |
    | **scalexy** | Scales the layout independently in the X and Y directions. |
    | **Voronoi** | Uses Voronoi regions to push nodes apart until overlaps are resolved. |

    **Layout Dimensions** (`dim=`) — sets the number of dimensions used when computing node positions. Higher values help the solver escape local minima; output is always projected back to 2D.

    **Rendering Dimensions** (`dimen=`) — specifies how many dimensions are used when interpreting node size attributes such as `width` and `height`.
  </Accordion>

  <Accordion title="neato options">
    **Overlap** — same options as `fdp` above.

    **Mode** — selects the position-solving algorithm:

    | Value | Description |
    |---|---|
    | **major** | Stress majorization — stable and widely used. |
    | **KK** | Kamada–Kawai spring model. |
    | **hier** | Top-down hierarchy-influenced layout. |
    | **ipsep** | Iterative penalty separation. |
    | **spring** | Classical spring-embedder. |
    | **maxent** | Maximum-entropy–inspired solver. |

    **Model** — selects how neato interprets edge relationships when computing ideal distances.

    **Layout Dimensions** and **Rendering Dimensions** — same behavior as `fdp` above.
  </Accordion>

  <Accordion title="sfdp options">
    **Overlap** — same options as `fdp` above.

    **Mode** — same values as `neato` above.

    **Smoothing** — post-processes raw node positions to reduce visual irregularities:

    | Value | Description |
    |---|---|
    | **none** | No smoothing. Uses raw layout positions. |
    | **avg_dist** | Adjusts positions based on average distances to neighbors. |
    | **graph_dist** | Smooths using graph-theoretic distances for global consistency. |
    | **power_dist** | Applies power-law weighting to distances. |
    | **rng** | Relative Neighborhood Graph–based smoothing. |
    | **spring** | Light spring-embedder pass. |
    | **triangle** | Triangle-based geometric smoothing for dense regions. |

    **Layout Dimensions** and **Rendering Dimensions** — same behavior as `fdp` above.
  </Accordion>

  <Accordion title="circo, osage, patchwork, twopi options">
    These engines have no additional dynamic layout options in the ribbon tab.
  </Accordion>
</AccordionGroup>

## Options group

The **Options** group provides graph-level, node-level, and edge-level checkboxes that include or exclude optional Graphviz attributes when the graph is generated.

### Graph options

| Control | Type | Description |
|---|---|---|
| **Center Drawing** | Checkbox | Centers the graph on the page when the graph is smaller than the page size. |
| **Force xlabel placement** | Checkbox | Places all `xlabel` attributes even when they overlap with nodes or other labels. |
| **Rotate 90 counterclockwise** | Checkbox | Rotates the final layout 90 degrees counterclockwise. |
| **Transparent Background** | Checkbox | Switches the graph background between white and transparent. Useful when pasting into image editors or Word documents. |
| **Include image path** | Checkbox | Adds the `imagepath` attribute to the graph source. |

### Node options

| Control | Type | Description |
|---|---|---|
| **Include stand-alone nodes** | Checkbox | When checked, retains nodes that have no edges in the diagram. When unchecked, removes isolated nodes. |
| **Include `Label`** | Checkbox | Includes or omits the `Label` column values in the graph. |
| **Include `External Label`** | Checkbox | Includes or omits the `External Label` (`xlabel`) column values. |
| **When the `Label` column is blank…** | Menu | Controls what is shown when a node has no label: either a blank label or the node's `Item` identifier. |

### Edge options

| Control | Type | Description |
|---|---|---|
| **Consolidate** | Checkbox | Consolidates multiple edge definitions between the same two nodes. |

## Data worksheet group

The **Data Worksheet** group provides menu items that control which columns and graphs are displayed on the `data` worksheet.

## Debug group

The **Debug** group helps you troubleshoot graph output.

| Control | Type | Description |
|---|---|---|
| **Verbose** | Toggle button | Appends the `-V` flag to the `dot` command so Graphviz emits detailed diagnostic messages. Output appears in the `console` worksheet. |
| **Console** | Toggle button | Shows or hides the `console` worksheet. |

<Tip>
Enable **Verbose** mode and open the `console` worksheet when a graph fails to render or produces unexpected results. The verbose output identifies missing attributes, unsupported values, and other issues that don't surface as visible errors.
</Tip>

## Help group

| Control | Type | Description |
|---|---|---|
| **Help** | Button | Opens the documentation page for the `data` worksheet in your browser. |
